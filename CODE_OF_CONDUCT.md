# Code of Conduct

Macster follows a straightforward contribution standard: be respectful, specific, and useful.

## Expected

- Keep feedback focused on the work.
- Explain risks with concrete examples.
- Prefer small, reviewable changes.
- Respect maintainers' scope decisions.

## Not Accepted

- Harassment or personal attacks.
- Publishing private information.
- Submitting code with secrets, local paths, IP addresses, or device-specific assumptions.
- Repeatedly pushing unrelated scope into small fixes.

Maintainers may remove comments, issues, pull requests, or contributors that violate these expectations.
